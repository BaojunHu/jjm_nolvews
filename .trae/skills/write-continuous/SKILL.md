# Write Continuous

## 目标
生成完整连贯的章节剧情概要，确保章节间平滑过渡，避免跳跃和缺失。

## 问题识别与解决

### 主要问题
1. **章节缺失**：批量生成时只生成关键章节，缺少中间过渡章节
2. **连续性差**：章节间跳跃太大，缺乏平滑过渡
3. **结构不完整**：没有完整的章节序列

### 解决方案
1. **强制完整生成**：确保指定范围内的所有章节都被生成
2. **章节间连贯性检查**：每章结尾与下一章开头保持逻辑衔接
3. **进度跟踪**：实时跟踪生成进度，确保不遗漏任何章节

## 执行步骤

### 1. 完整性验证
- 检查指定章节范围是否完整
- 确保不跳过任何中间章节
- 验证已存在章节的连贯性

### 2. 连贯性生成
- 每章结尾设置明确的下一章钩子
- 保持角色状态和关系的连续性
- 确保剧情发展的逻辑递进

### 3. 批量生成优化
- 按顺序逐章生成，避免跳跃
- 实时更新tracking文件
- 生成完成后验证章节完整性

## 使用示例

```bash
# 生成第1-35章，确保完整连贯
/write-continuous 1 --batch 35

# 继续生成后续章节
/write-continuous 36 --batch 35
```

## 输出质量保证

- **章节完整性**：确保指定范围内的所有章节都被生成
- **剧情连贯性**：章节间保持逻辑衔接和平滑过渡
- **角色一致性**：角色状态和关系发展保持连续
- **伏笔管理**：系统化埋设和回收伏笔

## 错误处理

- 如果章节缺失，自动补全
- 如果连贯性有问题，重新调整
- 如果tracking不一致，同步更新
